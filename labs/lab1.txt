
### Задание 1: Создание нового репозитория и добавление файла

1. **Создайте новый репозиторий на GitHub.**
    - Назовите репозиторий `my-first-repo`.
    - Добавьте описание: "Мой первый репозиторий на GitHub".

2. **Клонируйте репозиторий на ваш локальный компьютер.**
    ```sh
    git clone https://github.com/<ваше_имя_пользователя>/my-first-repo.git
    ```

3. **Создайте новый файл в клонированной папке.**
    - Назовите файл `README.md`.
    - Вставьте следующий текст в файл:
      ```markdown
      # Мой первый репозиторий

      Это мой первый опыт работы с GitHub!
      ```

4. **Добавьте и закоммитьте файл.**
    ```sh
    git add README.md
    git commit -m "Добавлен файл README.md"
    ```

    ```sh

### Задание 2: Работа с ветками и созданием Pull Request

1. **Создайте новую ветку для внесения изменений.**
    ```sh
    git checkout -b new-feature
    ```

2. **Внесите изменения в файл `README.md`.**
    - Откройте файл `README.md` и добавьте следующую строку:
      ```markdown
      ## Изменения

      Я добавил эту секцию для демонстрации работы с ветками и Pull Request на GitHub.
      ```

3. **Добавьте и закоммитьте изменения.**
    ```sh
    git add README.md
    git commit -m "Добавлен раздел изменений в README.md"
    ```

4. **Запушьте изменения на GitHub.**
    ```sh
    git push origin new-feature
    ```

5. **Создайте Pull Request для слияния изменений из новой ветки в ветку main.**
    - Перейдите на страницу вашего репозитория на GitHub.
    - Нажмите на кнопку "Compare & pull request".
    - Убедитесь, что выбранная база для слияния - `main`, а ветка для слияния - `new-feature`.
    - Добавьте описание для Pull Request (например, "Добавлен новый раздел в README.md").
    - Нажмите на кнопку "Create pull request".

6. **После проверки и одобрения Pull Request, выполните слияние изменений в ветку main.**
    - Нажмите на кнопку "Merge pull request".
    - Подтвердите слияние нажатием кнопки "Confirm merge".
